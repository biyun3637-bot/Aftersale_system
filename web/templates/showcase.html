工作原理

从客户消息到处理结果，每一步自动流转。鼠标悬停在节点上查看详情

💬客户消息
→
🧠意图分类LLM 或关键词匹配，识别客户意图
→
🔍订单查询Shopify API + 17Track 物流
↓
🤖RPA 兜底API 查不到时，自动启动浏览器抓取
→
⚖️决策引擎根据金额/物流/风险综合判定链路

1. 意图识别

系统分析客户消息，判断属于退款、订单异常还是售后问题，同时提取关联订单号。

2. 数据查询

调用 Shopify API 获取订单详情、17Track 查询物流追踪。接口失败时自动切换到 RPA 浏览器抓取。

3. 规则判定

根据金额、物流状态、风险等级综合判定处理方式：自动全额退款、部分退款或转人工审核。

4. 执行回复

自动调用 Shopify API 执行退款操作，同时根据处理结果生成客户回复内容。
